# App-wide i18n (Indonesian + English) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the entire kodapos UI translatable and switchable between Indonesian (`id`, source) and English (`en`), and establish Lingui macros as the convention for all new work.

**Architecture:** Lingui v6 (already configured). Both catalogs are imported statically. The active locale is read from `localStorage` (default `id`), activated at startup, and changed via a `LocaleProvider`/`useLocale()` hook whose `setLocale` calls `i18n.activate()` + persists; Lingui's `<I18nProvider>` re-renders translations. Strings are wrapped with `<Trans>`/`t` macros; data-driven labels (nav) use the `msg` macro + `i18n._()`. A scoped ESLint config with `eslint-plugin-lingui` enforces the convention.

**Tech Stack:** Lingui v6 (`@lingui/core`, `@lingui/react`, `@lingui/react/macro`, `@lingui/core/macro`), TanStack Start/Router, React 19, Vitest (edge-runtime), Playwright, ESLint (new, scoped to the lingui guard).

**Spec:** `docs/superpowers/specs/2026-05-29-i18n-multi-language-design.md`

```ts
// src/lib/locale.ts
export type Locale = 'id' | 'en';

export const LOCALES: { value: Locale; label: string }[] = [
  { value: 'id', label: 'Indonesia' },
  { value: 'en', label: 'English' },
];

export const DEFAULT_LOCALE: Locale = 'id';
const STORAGE_KEY = 'kodapos.locale';

export function normalizeLocale(value: string | null | undefined): Locale {
  return value === 'id' || value === 'en' ? value : DEFAULT_LOCALE;
}

/** Browser-only read; returns DEFAULT_LOCALE on the server. */
export function getStoredLocale(): Locale {
  if (typeof window === 'undefined') return DEFAULT_LOCALE;
  try {
    return normalizeLocale(window.localStorage.getItem(STORAGE_KEY));
  } catch {
    return DEFAULT_LOCALE;
  }
}

export function storeLocale(locale: Locale): void {
  if (typeof window === 'undefined') return;
  try {
    window.localStorage.setItem(STORAGE_KEY, locale);
  } catch {
    /* ignore (private mode, etc.) */
  }
}
```

- [ ] **Step 1: Create the provider**

```tsx
// src/components/locale-provider.tsx
'use client';

import { i18n } from '~/lib/i18n';
import { createContext, type ReactNode, useContext, useState } from 'react';
import { getStoredLocale, type Locale, storeLocale } from '~/lib/locale';

type LocaleContextValue = { locale: Locale; setLocale: (l: Locale) => void };
const LocaleContext = createContext<LocaleContextValue | null>(null);

export function LocaleProvider({ children }: { children: ReactNode }) {
  const [locale, setLocaleState] = useState<Locale>(() => getStoredLocale());

  function setLocale(next: Locale) {
    i18n.activate(next); // <I18nProvider> re-renders <Trans>/useLingui consumers
    storeLocale(next);
    setLocaleState(next);
  }

  return (
    <LocaleContext.Provider value={{ locale, setLocale }}>
      {children}
    </LocaleContext.Provider>
  );
}

export function useLocale(): LocaleContextValue {
  const ctx = useContext(LocaleContext);
  if (!ctx) throw new Error('useLocale must be used within a LocaleProvider');
  return ctx;
}
```

```ts
// src/lib/formater.ts
import { i18n } from '@lingui/core';

type DateFormat = 'day-month' | 'full';

function activeLocale(): string {
  return i18n.locale || 'id';
}

export function formatDate(value: string, format: DateFormat = 'day-month'): string {
  const d = new Date(value);
  if (Number.isNaN(d.getTime())) return value;
  const opts: Intl.DateTimeFormatOptions =
    format === 'full'
      ? { day: 'numeric', month: 'short', year: 'numeric' }
      : { day: 'numeric', month: 'short' };
  return new Intl.DateTimeFormat(activeLocale(), opts).format(d);
}

const idrFormatter = (locale: string) =>
  new Intl.NumberFormat(locale, {
    style: 'currency',
    currency: 'IDR',
    maximumFractionDigits: 0,
  });

export function formatIDR(amount: number): string {
  return idrFormatter(activeLocale()).format(amount);
}

export function formatIDRCompact(amount: number): string {
  if (amount >= 1_000_000)
    return `Rp${(amount / 1_000_000).toLocaleString(activeLocale(), { maximumFractionDigits: 1 })} jt`;
  if (amount >= 1_000)
    return `Rp${Math.round(amount / 1_000).toLocaleString(activeLocale())} rb`;
  return formatIDR(amount);
}
```

## Convention (document in PR / README)

All new UI strings use `<Trans>` / `t` (or `msg` + `i18n._()` for data-driven labels). After adding strings, run `pnpm lingui:extract`, translate `en`, and `pnpm lingui:compile`. `pnpm lint:i18n` enforces no unlocalized copy.
