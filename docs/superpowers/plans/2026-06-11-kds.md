# Kitchen Display System (KDS) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** A live kitchen board of order tickets (new → ready → done), auto-updating via reactive queries; sold orders tagged with their table.

**Architecture:** Orders gain `tableId` + `kitchenStatus`; `settleSale` sets `kitchenStatus:'new'` at paid. `convex/kitchen.ts` (`tickets` query over open-shift paid new/ready orders + `advance` mutation). `/kitchen` route renders the reactive board. `tableId` is threaded onto sold orders via `saleArgs`/`buildOrder` + the 4 payment dialogs (+ recall sets `currentTable`).
